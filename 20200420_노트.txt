배열 - 같은 타입의 여러 변수 
	
	배열을 사용하는 이유.
		
	같은 자료형의 다량의 데이터를 일괄로 다뤄야 할 때 편하게 할 수 있다.
	아래 예시와 같은 이유.
	
		ex) 철수의 1학기 성적
			
			int no = 14;		===>>		int[] students = {14, 78, 81, 76};		
			int math = 78;					철수의 학번 / 수학 / 영어 / 국어 점수를 
			int eng = 81;					하나의 배열변수로 처리 가능.
			int lang = 76;
			
	배열의 선언 방법
	
	int [] score = new int[5]; 5개의 int 값을 저장가능한 배열을 생성한다.
							   배열길이 [0 ~ 4] : 배열은 생성하면 0번부터 시작한다.
	특이사항 - String 타입(자바 제공 클래스)은 변수가 가지고 있는 값은 참조형 변수(메모리 주소 값)이다.
				String 타입의 배열은 참조형 주소값의 배열이다.
			   배열은 둘 이상의 데이터를 저장할 수 있는 형태의 인스턴스 이다
				
				
			   배열의 선언 -> 저장가능한 공간을 만드는 것이 아님.
							  주소값을 저장 가능한(참조 변수) 메모리를 생성함.
							  
							  선언단계 : 주소값 저장 가능한 공간을 생성.
							  int[] score; <- 참조변수
									ㄴ메모리주소 : 0x100
							  score = new int[5]; : 요소를 저장 가능한 공간(인스턴스 변수)을 생성(5개)
										                                      ㄴ 메모리주소 : 0x100, 0x101, 0x102, 0x103, 0x104 와 같은 연속된 메모리 주소(참조형).
										ㄴ 인스턴스 변수는 초기화 하지 않아도 초기 값이 생성된다. 자료형 마다 기본 초기화 되는 값이 있다.
	
	1차원 배열 : int[] ref = new int[5];
						▼		   ▼
					  참조변수    5개의 요소(메모리주소)를 저장 가능한 공간을 선언,
